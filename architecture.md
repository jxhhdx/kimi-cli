## 1. Tool 系统架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           TOOL SYSTEM (工具系统)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────┐     ┌─────────────────────────────────────────────┐   │
│  │   KimiToolset   │────▶│          Tool Registry (_tool_dict)         │   │
│  │                 │     │           {name: ToolType}                   │   │
│  └─────────────────┘     └─────────────────────────────────────────────┘   │
│           │                                                                  │
│           ├──▶ 内置工具 (Built-in Tools)                                     │
│           │     ├── ReadFile/WriteFile/StrReplaceFile (文件操作)            │
│           │     ├── Shell (命令执行)                                         │
│           │     ├── Grep/Glob (代码搜索)                                     │
│           │     ├── WebSearch/FetchURL (网络)                                │
│           │     ├── Task (子代理)                                            │
│           │     ├── Todo (任务列表)                                          │
│           │     └── Think (思考工具)                                         │
│           │                                                                  │
│           └──▶ MCP 外部工具 (Model Context Protocol)                         │
│                 ├── MCPServerInfo {status, client, tools[]}                  │
│                 └── MCPTool {server_name, mcp_tool, client}                  │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                     工具调用流程 (Tool Call Flow)                    │   │
│  │                                                                     │   │
│  │   LLM ──▶ ToolCall ──▶ KimiToolset.handle() ──▶ 找到对应工具        │   │
│  │                                              ──▶ 解析参数            │   │
│  │                                              ──▶ 执行 __call__()     │   │
│  │                                              ──▶ 返回 ToolResult     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                     依赖注入 (Dependency Injection)                  │   │
│  │                                                                     │   │
│  │   tool_deps = {                                                     │   │
│  │       KimiToolset: toolset,         # 自引用                        │   │
│  │       Runtime: runtime,             # 运行时环境                    │   │
│  │       Config: runtime.config,       # 配置                          │   │
│  │       Session: runtime.session,     # 会话                          │   │
│  │       DenwaRenji: ...,              # D-Mail 系统                   │   │
│  │       Approval: ...,                # 审批系统                      │   │
│  │       LaborMarket: ...,             # 子代理市场                    │   │
│  │   }                                                                 │   │
│  │                                                                     │   │
│  │   通过 _load_tool() 自动注入构造函数参数                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Tool 核心类

```python
# KimiToolset - 工具集管理器
class KimiToolset:
    _tool_dict: dict[str, ToolType]        # 工具注册表
    _hidden_tools: set[str]                # 对LLM隐藏的工具
    _mcp_servers: dict[str, MCPServerInfo] # MCP服务器
    
    def add(tool: ToolType)                # 注册工具
    def handle(tool_call: ToolCall)        # 执行工具调用
    def load_tools(paths, dependencies)    # 加载内置工具
    async def load_mcp_tools(configs)      # 加载MCP工具

# 内置工具基类 (使用 kosong 库)
class ReadFile(CallableTool2[Params]):     # Pydantic 参数模型
    async def __call__(self, params) -> ToolReturnValue

# MCP 工具包装器
class MCPTool(CallableTool):
    async def __call__(self, **kwargs) -> ToolReturnValue
```

## 2. Memory (Context) 系统架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         MEMORY SYSTEM (记忆系统)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                         Context (上下文)                             │   │
│  │                                                                     │   │
│  │   _history: list[Message]          # 对话历史 (内存)                 │   │
│  │   _token_count: int                # 当前token数                    │   │
│  │   _file_backend: Path              # 持久化文件 (context.jsonl)      │   │
│  │   _next_checkpoint_id: int         # 下一个checkpoint ID            │   │
│  │                                                                     │   │
│  │   方法:                                                             │   │
│  │   ├── restore()        # 从文件恢复历史                             │   │
│  │   ├── append_message() # 添加消息并持久化                           │   │
│  │   ├── checkpoint()     # 创建检查点                                 │   │
│  │   ├── revert_to(id)    # 回滚到指定检查点 (时间旅行!)                │   │
│  │   ├── clear()          # 清空上下文                                 │   │
│  │   └── update_token_count()                                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    文件格式 (context.jsonl)                          │   │
│  │                                                                     │   │
│  │   {"role": "user", "content": [...]}                                │   │
│  │   {"role": "assistant", "content": [...], "tool_calls": [...]}      │   │
│  │   {"role": "tool", "tool_call_id": "...", "content": [...]}         │   │
│  │   {"role": "_usage", "token_count": 1234}                           │   │
│  │   {"role": "_checkpoint", "id": 0}                                  │   │
│  │   ...                                                               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    DenwaRenji (D-Mail 系统)                          │   │
│  │                                                                     │   │
│  │   DMail {message, checkpoint_id}                                    │   │
│  │                                                                     │   │
│  │   send_dmail(dmail)         # 发送消息到过去                        │   │
│  │   fetch_pending_dmail()     # 获取待处理的D-Mail                    │   │
│  │                                                                     │   │
│  │   用途: 子代理通过 D-Mail 向父代理的过去发送消息                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    Compaction (上下文压缩)                           │   │
│  │                                                                     │   │
│  │   SimpleCompaction                                                  │   │
│  │   ├── max_preserved_messages = 2     # 保留最近2条消息               │   │
│  │   └── compact(messages, llm)                                        │   │
│  │       ├── 准备: 分割为 to_compact + to_preserve                     │   │
│  │       ├── 调用 LLM 生成摘要                                          │   │
│  │       └── 返回: [摘要消息] + [保留消息]                              │   │
│  │                                                                     │   │
│  │   触发条件 (should_auto_compact):                                    │   │
│  │   token_count >= max_context * trigger_ratio                        │   │
│  │   OR token_count + reserved >= max_context                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 3. 核心循环架构 (KimiSoul)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      KIMI SOUL (核心代理循环)                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────┐     ┌─────────────────────────────────────────────┐   │
│  │    KimiSoul     │────▶│              Agent Loop                     │   │
│  │                 │     │                                             │   │
│  │  _agent         │     │   while True:                               │   │
│  │  _context       │     │       1. Check & Trigger Compaction         │   │
│  │  _runtime       │     │       2. Create Checkpoint                  │   │
│  │  _compaction    │     │       3. Run Step (kosong.step)             │   │
│  │  _steer_queue   │     │       4. Execute Tool Calls                 │   │
│  │                 │     │       5. Handle D-Mail (BackToTheFuture)    │   │
│  └─────────────────┘     │       6. Check Stop Conditions              │   │
│                          │                                             │   │
│                          │   Stop Reasons:                             │   │
│                          │   - "no_tool_calls": LLM 直接回复           │   │
│                          │   - "tool_rejected": 用户拒绝工具调用       │   │
│                          └─────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                         Step 流程                                    │   │
│  │                                                                     │   │
│  │   ┌─────────────┐    ┌─────────────┐    ┌─────────────────────┐   │   │
│  │   │  Dynamic    │───▶│  kosong.    │───▶│  Wait Tool Results  │   │   │
│  │   │  Injection  │    │  step()     │    │                     │   │   │
│  │   └─────────────┘    └─────────────┘    └─────────────────────┘   │   │
│  │          │                  │                      │               │   │
│  │          ▼                  ▼                      ▼               │   │
│  │   <system-reminder>    LLM Call              并行执行工具           │   │
│  │   动态注入系统提示     流式响应               收集结果                │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    关键特性: Checkpoint & Revert                     │   │
│  │                                                                     │   │
│  │   1. 每次 Step 前创建 checkpoint                                     │   │
│  │   2. 收到 D-Mail 时 raise BackToTheFuture                            │   │
│  │   3. 捕获异常后 revert_to(checkpoint_id)                             │   │
│  │   4. 注入 D-Mail 消息后继续                                          │   │
│  │                                                                     │   │
│  │   实现效果: "时间旅行" - 子代理可以改变父代理的过去                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. 关于 RAG

## 6. 关键设计亮点

| 特性 | 实现 | 说明 |
|------|------|------|
| **Checkpoint/Revert** | Context + DenwaRenji | 支持"时间旅行"，子代理可以修改父代理的过去 |
| **Compaction** | SimpleCompaction | 上下文压缩，保留最近2条消息，其余LLM摘要 |
| **依赖注入** | _load_tool() | 通过构造函数参数类型自动注入依赖 |
| **MCP 支持** | fastmcp | 外部工具通过 Model Context Protocol 接入 |
| **Steer 机制** | steer_queue | 用户可以在执行中注入实时指令 |
| **Plan Mode** | plan_mode flag | 只读模式，禁止文件修改工具 |
