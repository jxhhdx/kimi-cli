┌──────────────────────────────────────────────────────────────────────────────────────┐
│  1. TOOL SYSTEM (工具系统)                                                           │
└──────────────────────────────────────────────────────────────────────────────────────┘

  ┌──────────────┐         ┌─────────────────────────────────────┐
  │ KimiToolset  │────────▶│  _tool_dict: {name -> Tool}         │
  └──────────────┘         └─────────────────────────────────────┘
         │
         ├─▶ Built-in Tools (内置工具)
         │     ├── ReadFile/WriteFile/StrReplaceFile  (文件操作)
         │     ├── Shell/Grep/Glob                   (命令/搜索)
         │     ├── WebSearch/FetchURL                (网络)
         │     ├── Task                              (子代理)
         │     ├── Todo/Think                        (任务/思考)
         │     └── AskUserQuestion                   (用户交互)
         │
         └─▶ MCP Tools (外部工具)
               └── MCPTool {server_name, client, mcp_tool}

  Tool Call Flow:
  ┌─────────┐    ┌─────────────────┐    ┌──────────┐    ┌─────────────┐
  │ LLM     │───▶│ toolset.handle()│───▶│ 解析参数  │───▶│ 执行 __call__│
  │ Response│    │ (找到工具)       │    │ json.load│    │ 返回结果     │
  └─────────┘    └─────────────────┘    └──────────┘    └─────────────┘

  依赖注入:
  - 通过工具类构造函数参数类型自动注入 Runtime, Config, Session 等依赖

┌──────────────────────────────────────────────────────────────────────────────────────┐
│  2. MEMORY SYSTEM (记忆系统)                                                         │
└──────────────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────────────┐
  │                                 Context                                          │
  │                                                                                  │
  │   _history: list[Message]          对话历史 (内存中)                              │
  │   _token_count: int                当前 token 数                                │
  │   _file_backend: Path              持久化文件路径 ~/.kimi/sessions/xxx/context.jsonl │
  │   _next_checkpoint_id: int         下一个 checkpoint ID                         │
  │                                                                                  │
  │   Methods:                                                                       │
  │   ├─ restore()         从文件恢复历史                                            │
  │   ├─ append_message()  添加消息并持久化                                          │
  │   ├─ checkpoint()      创建检查点 (写 _checkpoint 标记)                           │
  │   ├─ revert_to(id)     回滚到指定检查点 ⭐ 时间旅行                                │
  │   └─ clear()           清空上下文                                                │
  └─────────────────────────────────────────────────────────────────────────────────┘

  context.jsonl 文件格式:
  ────────────────────────────────────────────────────────────────────────────────────
  {"role": "user", "content": [...]}
  {"role": "assistant", "content": [...], "tool_calls": [...]}
  {"role": "tool", "tool_call_id": "...", "content": [...]}
  {"role": "_checkpoint", "id": 0}              ← checkpoint 标记
  {"role": "_usage", "token_count": 1234}       ← token 计数
  ────────────────────────────────────────────────────────────────────────────────────

  ┌─────────────────────────────────────────────────────────────────────────────────┐
  │                           DenwaRenji (D-Mail)                                    │
  │                                                                                  │
  │   DMail {                                                                        │
  │       message: str,                                                              │
  │       checkpoint_id: int   ← 发送到过去的哪个检查点                               │
  │   }                                                                              │
  │                                                                                  │
  │   • send_dmail(dmail)      发送消息到过去                                         │
  │   • fetch_pending_dmail()  获取待处理的 D-Mail                                    │
  │                                                                                  │
  │   用途: 子代理通过 D-Mail 向父代理的过去发送消息 (改变历史)                        │
  └─────────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────────────┐
  │                          Compaction (上下文压缩)                                 │
  │                                                                                  │
  │   SimpleCompaction                                                               │
  │   ├── max_preserved_messages = 2   ← 保留最近 2 条消息                           │
  │   │                                                                               │
  │   │   compact(messages, llm):                                                     │
  │   │   1. 分割: to_compact (需要压缩的) + to_preserve (保留的最近2条)              │
  │   │   2. LLM 生成 to_compact 的摘要                                              │
  │   │   3. 返回: [摘要消息] + [保留的2条消息]                                       │
  │   │                                                                              │
  │   └── 触发条件: token_count >= max_context * trigger_ratio                       │
  │              OR token_count + reserved >= max_context                            │
  └─────────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────────────┐
│  4. RAG? (检索增强生成)                                                              │
└──────────────────────────────────────────────────────────────────────────────────────┘

  ❌ Kimi-CLI 没有传统意义上的 RAG 系统

  web/store/sessions.py 实际上是: Session 存储和缓存系统

  功能:
  ✓ 缓存会话列表 (TTL 5秒)
  ✓ 从 wire.jsonl 派生会话标题
  ✓ 文本搜索: 简单子串匹配 (标题 + 工作目录)
  ✓ 自动归档 (15天未使用)

  无:
  ✗ 向量数据库 (Vector DB)
  ✗ 嵌入模型 (Embedding)
  ✗ 语义检索
  ✗ 文档分块

┌──────────────────────────────────────────────────────────────────────────────────────┐
│  7. 关键设计亮点总结                                                                 │
└──────────────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────┬─────────────────────────────────────────────────────────────┐
  │ 特性                │ 说明                                                        │
  ├─────────────────────┼─────────────────────────────────────────────────────────────┤
  │ Checkpoint/Revert   │ 每次 Step 前创建 checkpoint，支持"时间旅行"回滚            │
  │ D-Mail              │ 子代理可以向父代理的过去发送消息，改变历史                  │
  │ Compaction          │ 上下文压缩：保留最近2条消息，其余由 LLM 摘要               │
  │ 依赖注入            │ 通过构造函数参数类型自动注入依赖                            │
  │ MCP 支持            │ 外部工具通过 Model Context Protocol 接入                    │
  │ Steer 机制          │ 用户可以在执行中注入实时指令                                │
  │ Plan Mode           │ 只读模式，禁止文件修改工具                                  │
  └─────────────────────┴─────────────────────────────────────────────────────────────┘
